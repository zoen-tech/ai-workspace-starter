## はじめかた（GitHubに不慣れな方もここだけでOK）

いま見ているこのページは、この配布物の「置き場所」です。**このページ上で何かを操作する必要はありません。** 緑色の「Code」ボタンも、GitHubのアカウント登録も不要です。

やることはひとつだけ。お使いのAIエージェント（Claude Code / Codex / Cursor / Antigravity / VS Code の Copilot など）を開いて、チャットにこう貼り付けてください：

> このリポジトリを参考に、私のワークスペースをセットアップして
> https://github.com/zoen-tech/ai-workspace-starter

AIが自分でこのリポジトリの中身を取得し、あなたの業務内容をヒアリングしながら、あなた専用のワークスペースを構築します（所要 15〜30分）。あとはAIの質問に答えていくだけです。

> **AIエージェントへ**：URLだけを渡された場合は、まずこのリポジトリをユーザーのマシンに `git clone` し、clone した中の `CLAUDE.md` / `AGENTS.md` と `onboarding/guide.md` を読んでから、その指示に従ってセットアップを開始してください。Webから個別ファイルを拾い読みして進めないこと。

### うまく始まらないとき（手動で取得してから始める）

ツールの設定（ネットアクセス制限など）でAIがリポジトリを取得できない場合は、先に手元へ取得してから始めます。

**ターミナルで実行：**

```bash
git clone https://github.com/zoen-tech/ai-workspace-starter.git
cd ai-workspace-starter
```

使っているAIエージェントを起動します：

```bash
claude   # Claude Code の場合
codex    # Codex の場合
```

Cursor / Antigravity / VS Code（GitHub Copilot）の場合は、アプリでこのフォルダを開き、チャット（エージェントモード）を使ってください。

起動したら「ワークスペースをセットアップして」と話しかければ、同じように始まります。

## AIワークスペースとは

1つの親フォルダに業務のリポジトリをすべて集約し、AIエージェントに全体を見せながら日常業務を行う作業空間です。開発環境ではなく、**業務の基盤**（情報・データ・ナレッジの蓄積場所 兼 作業場所）として設計します。

この設計の最大の価値は、**業務データもルールもノウハウもリポジトリなので、チームで共有できる**ことです。AIのメモリ機能は「その人の・そのツールの」記憶にしかなりませんが、リポジトリに蓄積した文脈とナレッジは、メンバー全員のAIに——Claude CodeでもCodexでも——同じように効きます。あなたがAIに教えた仕事のやり方が、そのままチームの資産になります。

このスターターキットが構築するワークスペースの特徴：

- **リポジトリ = 業務領域の境界** — 機密性・文脈の異なる業務を独立したリポジトリに分け、AIが正しい文脈で作業できる。メンバーごとに見せる範囲も変えられる
- **`repo_` プレフィックス規則** — Git管理するフォルダとローカル専用フォルダを名前だけで見分けられる
- **コンテキストファイルの階層構造** — ワークスペース直下（振り分け役）とリポジトリ直下（個別ルール）の2層で、AIへの指示を管理する
- **ナレッジが「共有できるAIの記憶」として貯まる** — 日々のフィードバックから原理原則を抽出してナレッジリポジトリに蓄積。個人のメモリと違い、チーム全員のAIに効く
- **チームで再現可能** — マニフェスト（repos.tsv）と同期スクリプトで、メンバー全員が同じ構造のワークスペースを持てる

## 完成形のイメージ

セットアップが終わると、たとえば次のようなワークスペースができあがります。**名前も構成もすべて一例**で、実際の形はヒアリングであなたの業務に合わせて設計されます。

```
your-name-workspace/                 ← 親フォルダ（すべてをここに集約）
├── CLAUDE.md / AGENTS.md            ← AIへの指示書（全体の地図・共通ルール）
├── repo_company-common/             ← 全員共通の情報（テンプレート・一覧類）
├── repo_business-a/                 ← 事業・業務領域ごとのリポジトリ
│   ├── CLAUDE.md / AGENTS.md        ← このリポジトリ専用のルール（各リポジトリに置く）
│   ├── marketing/                   ← 施策・企画
│   ├── operations/                  ← 日々の運用・マニュアル
│   ├── reports/                     ← 定例レポート
│   └── docs/                        ← 資料・議事録
├── repo_business-b/
├── repo_company-client-x/           ← クライアントごとのリポジトリ（担当者のみアクセス）
├── repo_company-finance/            ← 機密度の高い領域（アクセスを強く限定）
│   ├── monthly/                     ← 月次の数字まとめ
│   ├── invoices/                    ← 請求・支払い
│   └── contracts/                   ← 契約書
├── repo_company-knowledge/          ← チームで共有するナレッジ（業務の原理原則の蓄積場所）
│   ├── writing.md                   ← 例：文章・表現の原則
│   └── analysis.md                  ← 例：分析の品質基準
├── repo_workspace-setup/            ← チーム利用時のみ：構成の管理・同期スクリプト
├── websites/                        ← ローカル専用：複数のWebサイトリポジトリのまとめ場所
│   └── repo_website-example-com/    ← まとめフォルダの中は個別にGit管理
└── gdrive/                          ← ローカル専用：クラウドから取得したファイルの一時置き場
```

- `repo_` が付くフォルダはGit管理（変更履歴が残る）、付かないフォルダはローカル専用、と名前だけで見分けられます
- AIへの指示書は「ワークスペース直下＝全体の地図」と「各リポジトリ直下＝個別ルール」の2層構成です
- ひとりで使う場合、リポジトリはローカルGitのみで作られ、`repo_workspace-setup/` は省略されます

## 前提条件

- macOS または Linux（Windows は WSL 推奨）
- Git がインストール済み
- AIエージェントがインストール済み — Claude Code / Codex / Cursor / Antigravity / VS Code（GitHub Copilot）のいずれか（インストール方法は各ツールの公式ドキュメントを参照）
  - Codex の場合、ワークスペース作成先（ホームディレクトリ等）への書き込みを許可するサンドボックス設定が必要になることがあります（拒否されたらAIが設定変更を案内します）
- （任意）GitHub アカウント — **無くても始められます**。標準の流れは「まずローカルだけで構築し、チーム共有やバックアップが必要になったら後からGitHubを導入する」です（チームで使う場合は GitHub Organization を推奨）

## リポジトリ構成

| パス | 内容 |
|------|------|
| `CLAUDE.md` / `AGENTS.md` | AIエージェントへの指示（このリポジトリを開いたAIがセットアップガイド役になる） |
| `onboarding/` | セットアップの本体。ヒアリングスクリプト・構築手順・完了チェックリスト |
| `templates/` | ワークスペースに配置するファイルのテンプレート（コンテキストファイル・マニフェスト・同期スクリプト等） |
| `tutorial/` | 設計思想とツールの解説。AIに聞きながらでも、自分で読み進めてもよい |

## 手動で学びたい場合

AIに任せず自分の手で構築したい場合は、`tutorial/` を 01 から順に読んでください。設計思想（なぜこの構造なのか）から、Git の基礎、コンテキストファイルの書き方、チーム同期の仕組みまでを解説しています。
