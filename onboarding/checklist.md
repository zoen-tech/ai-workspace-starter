# 完了チェックリスト（AIエージェント向け）

構築後、以下をすべて確認してからセットアップ完了とする。確認できない項目は理由をユーザーに報告する。

## 構造

- [ ] ワークスペース直下にコンテキストファイル（CLAUDE.md / AGENTS.md）があり、リポジトリ一覧表が実際のフォルダ構成と一致している
- [ ] すべての `repo_` フォルダに `.git` があり、初回コミットが済んでいる
- [ ] ローカル専用フォルダ（プレフィックスなし）が構成ルール表に記載されている

## Git / GitHub（利用時のみ）

- [ ] 各リポジトリが push 済みで、GitHub 上で確認できる
- [ ] 機密度の高いリポジトリが private になっている
- [ ] （チーム利用時）`repos.tsv` の内容が実際のリポジトリ構成と一致している
- [ ] （チーム利用時）`sync-repos.sh --dry-run` がエラーなく動き、出力が期待どおり

## 動作確認

- [ ] ワークスペース直下でAIエージェントを起動し直し、「リポジトリ一覧を教えて」と質問すると、コンテキストファイルに基づいて正しく答える
- [ ] いずれかのリポジトリでの作業を依頼すると、そのリポジトリ直下のコンテキストファイルを読んでから作業を始める
- [ ] （ナレッジリポ作成時）試しにフィードバックを1つ与え、ナレッジリポジトリの該当ファイルに原理原則として追記されることを確認した

## 引き渡し

- [ ] `tutorial/` の場所と読み進め方を案内した
- [ ] `setup-notes.md` の扱い（削除するか残すか）を確認した
- [ ] 最初の業務をこのワークスペースで始める提案をした
- [ ] （ローカルのみ運用時）バックアップが無い状態であることと、GitHub導入のタイミング・移行方法を伝えた
