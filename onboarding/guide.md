# セットアップガイド（AIエージェント向け指示書）

あなたの役割：ユーザーにヒアリングしながら、ユーザー専用の「AIワークスペース」をユーザーのマシン上に構築すること。

## 全体の流れ

1. **ヒアリング** — `interview.md` の質問スクリプトに従って聞き取る
2. **設計提示** — 回答をもとに、構築するワークスペースの全体像（フォルダ構成図・作成するファイルの一覧）を提示し、承認を得る
3. **構築** — `build-steps.md` の手順で実行する
4. **確認** — `checklist.md` で動作確認する
5. **チュートリアル案内** — `tutorial/` の読み進め方を案内し、最初の業務をこのワークスペースで始める方法を1つ提案する

## 進め方のルール

- 質問は一度に1〜3個まで。まとめて大量に聞かない
- 選択肢は数字（1/2/3）で提示し、必ず推奨を1つ添える
- 回答が曖昧なときは、推奨案を1つ添えて確認する。勝手に想像で埋めない
- ファイルやフォルダを作り始める前に、必ず全体像の承認を得る（流れの2）
- **このリポジトリ（ai-workspace-starter）は編集しない。** 構築はすべてユーザーのワークスペースフォルダ側に行う
- GitHubリポジトリの作成・push など外部に影響する操作は、実行前に1件ずつ内容を伝えて確認する
- ユーザーが非エンジニアである可能性を常に考慮する。専門用語（リポジトリ・コミット等）には初出時に一言説明を添える
- ユーザーに手動操作を依頼するのは、AIが代行できないもの（アカウント作成・認証ログイン等）だけにする。コマンドで済むものはAIが実行する

## 中断と再開

- ヒアリングの回答は、ワークスペースフォルダ直下の `setup-notes.md` に記録しながら進める（フォルダ作成前はメモリ上でよいが、フォルダを作ったら最初に書き出す）
- セッションが途切れた場合に備え、`setup-notes.md` には「完了したステップ／次にやること」も記録する
- 再開を依頼されたら、まず `setup-notes.md` を読んで続きから進める
- `setup-notes.md` はセットアップ完了後に削除してよいか確認し、不要なら削除する

## トラブル時

- コマンドの失敗は原因を特定してから再試行する。同じコマンドの単純リトライを繰り返さない
- GitHub の権限・認証エラーは `gh auth status` の確認から始める
- Codex でワークスペースフォルダ（このリポジトリの外）への書き込みが拒否される場合は、サンドボックス設定が原因。ユーザーに書き込み許可の設定変更（または該当フォルダを許可対象に追加）を案内する
- 解決できない問題はユーザーに状況・原因・選択肢（推奨付き）を簡潔に伝える
