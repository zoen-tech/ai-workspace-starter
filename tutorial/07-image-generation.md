# 07. AI画像生成 — 「描かせる画像」と「組ませる画像」を分ける

> **要旨：** 業務で使う画像は2種類に分かれる。デザイン性が要る画像（バナー・図解・挿絵）は画像生成AIに描かせ、データを正確に並べる画像（価格表・一覧表）はAIにコードで組ませる。この使い分けと、検品・再現性のルールをワークスペースに定めておくと、画像づくりが安定した定型業務になる。

> **Note:** この章は応用編です。実装例としてClaude Code＋Codex CLI（ChatGPTの画像生成）の組み合わせを使いますが、考え方自体はツールに依存しません。

---

## なぜ使い分けるのか

コーディングエージェント（Claude Code等）は「絵」を描けません。できるのはコードで図形や文字を配置することだけです。逆に、画像生成AI（ChatGPTのimage 2.0等）は魅力的なビジュアルを描けますが、**文字や数字を崩すことがある**という弱点があります。

この特性の違いから、画像は2種類に分けて作り方を変えます。

| 画像の種類 | 作り方 | 理由 |
|-----------|--------|------|
| バナー・図解・挿絵など、デザイン性が要るもの | **画像生成AIに描かせる** | コード描画ではデザイン品質が出ない |
| 価格表・料金表・一覧表など、データを正確に並べるだけのもの | **AIにコード（Python/PIL等）で組ませる** | 1文字も崩れてはいけない。生成AIは信頼できない |
| 中間（デザインもデータ正確性も要る） | 画像生成AIでよいが、**生成物を元データと全項目照合**してから使う | 崩れの検出を人（またはAI）の検品で担保する |

コードで組む場合の重要ルールがひとつ：**数値をスクリプトにベタ書きせず、元データのファイル（料金表のMarkdown等）から自動抽出させる**こと。価格改定のたびに2箇所を直す二重管理を防ぎ、「元データを直してスクリプトを再実行すれば画像が最新になる」状態を保ちます。

## デザイン画像のフロー（実装例）

コーディングエージェントから画像生成AIを呼び出す構成にすると、「バナーを作って」の一言でフロー全体が回ります。Claude Code＋Codex CLIの例では次の3段構成です。

```
① Claude：デザイン指示書（プロンプト）を作成
        ↓
② Codex CLI：ChatGPTのimage 2.0ツールに描画を依頼（数分〜十数分）
        ↓
③ Claude：検品（文言・数値の全項目照合＋構図チェック)
        ↓
④ 必要に応じて後合成（ロゴ・キャラは公式素材をスクリプトで貼る）
```

事前準備はCodex CLIのインストールとログインだけです。

```bash
npm install -g @openai/codex
codex login
```

実行コマンドの型：

```bash
codex exec --skip-git-repo-check --sandbox workspace-write "image_gen ツールで以下の画像を生成し、<保存先パス>に保存して。<デザイン指示>"
```

生成には数分〜十数分かかるため、エージェントから呼ぶ場合はバックグラウンド実行にしてファイル生成を待ちます。また、エージェントのシェルがサンドボックス化されている場合、外部APIへのDNS解決がブロックされて失敗することがあります（その場合はサンドボックスを外して実行します）。

## プロンプトに必ず入れる4要素

画像生成のプロンプトは「デザイン指示書」として構造化します。

1. **画像仕様** — サイズ（px）・用途・トーン・配色（基調色＋アクセント色の数まで指定）
2. **レイアウト** — 領域を%で区切って要素を配置指示。視線導線（Z型等）とセーフマージン（外周60px目安）
3. **掲載文言** — 一字一句そのまま書く。「指定したテキスト以外の文字は入れない」と明記
4. **制約** — 「日本語の文字は一字一句正確に、字形を崩さず描画。文字化け・豆腐厳禁」「テキストは帯・カード・ボタンの内側に完全に収める」など

## 品質を守る3つの決まり

### 1. 検品は2観点で行う

- **文言・数値の正確性** — 元データと全項目を照合する。日本語は特定の字が崩れやすく、崩れた字は**言い換えで回避**するのが実用的です（例：「お会計」の「会」が欠落グリフになったため「お支払い」に変更）
- **構図** — 枠内収まり・空白の死に・要素の重なり・見切れ。後合成がある場合は合成後の完成状態で再チェック

### 2. ロゴ・キャラクターはAIに描かせない

会社ロゴやマスコットキャラなど「1ピクセルも崩れてはいけない要素」は、生成画像に含めず**公式素材のPNGを後からスクリプトで合成**します（リサイズのみ・無加工）。プロンプト側では「キャラクターは描かない」＋空きスペースの位置とサイズ（例：左下 幅35%×高さ30%）を指定しておきます。

### 3. 再現手順を必ず一緒に残す

生成した画像は、**再生成プロンプト（または生成スクリプト）とセットで**リポジトリに格納します。後から価格や文言が変わったとき、プロンプトの該当箇所だけ直して再生成できる状態が合格ラインです。これは03章のコンテキストファイル、04章のナレッジと同じ発想です——成果物だけでなく「作り方」をリポジトリに残すことで、チームの誰のAIでも同じ品質で再現できます。

この使い分けルールをワークスペースのコンテキストファイル（CLAUDE.md / AGENTS.md）に書いておけば、以降は「〜のバナーを作って」「料金表の画像にして」と言うだけで、AIが適切な作り方を選んで検品まで実行するようになります。
